### RAG 检索策略回滚到 DeepSeek 规划

- 发现“演示二叉树的后序遍历”曾被上一轮“顺序表”相关 RAG 关键词带偏，回答为“知识库资料不足”。
- 临时尝试过本地主题过滤表，但这违背了既定策略，已撤掉。
- 当前策略改为：
  - 用户提问后，先让 DeepSeek 输出本轮 RAG 检索计划。
  - 检索计划包含：
    - `query_keywords`
    - `groups`
    - 每组 `weight`
    - 仍相关的 `history_keywords`
  - 当前问题关键词权重最高。
  - DeepSeek 分组关键词按组权重参与检索。
  - 历史 RAG 关键词只有在 DeepSeek 判断仍相关时，才以低权重辅助。
- `rag.py` 增加 `boost_weights`，让权重直接进入关键词检索评分。
- 不再使用本地 `RAG_TOPIC_GROUPS` / `_matched_rag_topics` / `filter_rag_keywords_for_question` 这类本地主题判断。
