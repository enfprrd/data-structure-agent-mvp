# 树与二叉树

> 根据教材第 6 章整理的学习提纲，用于本项目 RAG 检索，不复制教材原文。

## 树的概念

树是由结点组成的有限集合。非空树有且仅有一个根结点，其余结点可分为若干互不相交的子树。

树形结构体现一对多关系：除根结点外，每个结点只有一个双亲；一个结点可以有多个孩子。

基本术语包括根、结点、边、孩子、双亲、兄弟、祖先、子孙、叶子、度、层次、高度、森林。

## 二叉树概念

二叉树是每个结点最多有两棵子树的树，子树有左右之分，次序不能随意交换。

二叉树可以为空。二叉树不是“度为 2 的树”的简单等同，因为二叉树的左右孩子有明确位置，即使只有一个孩子也要区分左孩子还是右孩子。

## 二叉树性质

常用性质：
- 第 `i` 层最多有 `2^(i-1)` 个结点。
- 深度为 `k` 的二叉树最多有 `2^k - 1` 个结点。
- 对任意非空二叉树，若叶子结点数为 `n0`，度为 2 的结点数为 `n2`，则 `n0 = n2 + 1`。

满二叉树每一层结点数都达到最大。完全二叉树按层从左到右连续编号，适合用数组顺序存储。

## 二叉树存储

顺序存储适合完全二叉树。若从 1 开始编号，结点 `i` 的左孩子为 `2i`，右孩子为 `2i + 1`，双亲为 `i / 2` 向下取整。

链式存储常用二叉链表，每个结点含数据域、左孩子指针和右孩子指针。

```c
typedef struct BiTNode {
    int data;
    struct BiTNode *lchild, *rchild;
} BiTNode, *BiTree;
```

## 二叉树遍历

遍历是按某种规则访问树中每个结点一次且仅一次。

深度优先遍历：
- 先序：根、左、右。
- 中序：左、根、右。
- 后序：左、右、根。

广度优先遍历是层序遍历，通常借助队列。

递归遍历要有空树终止条件。非递归先序、中序遍历常借助栈。

## 线索二叉树

普通二叉链表中有许多空指针。线索二叉树利用空指针保存遍历序列中的前驱或后继信息。

建立线索时通常要区分指针域指向孩子还是线索，因此需要标志位，例如 `ltag`、`rtag`。

## 哈夫曼树

哈夫曼树又称最优二叉树，常用于构造最短带权路径长度的编码树。

构造思想：每次选权值最小的两棵树合并为一棵新树，新树权值为两者之和，直到只剩一棵树。

哈夫曼编码特点：任一字符编码都不是另一个字符编码的前缀，因此可以唯一解码。

## 易错点

- 把二叉树误认为每个结点必须有两个孩子。
- 混淆满二叉树和完全二叉树。
- 遍历序列中根的位置判断错误：先序根在前，中序根在中，后序根在后。
- 层序遍历忘记使用队列。
- 线索二叉树中不区分孩子指针和线索指针。
- 哈夫曼树构造时没有每次选择当前最小的两项。
