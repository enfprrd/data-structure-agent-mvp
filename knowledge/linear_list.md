# 线性表

> 说明：本文件是面向 RAG 的本科数据结构学习摘要，不复制教材原文。后续可以继续按教材小节补充例题、图示和课堂要求。

## 线性表概念

线性表是由相同类型数据元素组成的有限序列。若记为 `L = (a1, a2, ..., an)`，其中 `n` 为线性表长度，`n = 0` 时称为空表。

线性表的逻辑特点：

- 元素之间是一对一的线性关系。
- 第一个元素没有直接前驱。
- 最后一个元素没有直接后继。
- 除第一个元素外，其余每个元素有且仅有一个直接前驱。
- 除最后一个元素外，其余每个元素有且仅有一个直接后继。

线性表关注的是逻辑结构，不直接限定物理存储方式。它可以用顺序存储实现，也可以用链式存储实现。

## 线性表基本操作

线性表常见基本操作包括：

- 初始化：建立一个空表。
- 判空：判断表中是否没有元素。
- 求长度：统计当前元素个数。
- 遍历：按顺序访问每个元素。
- 按位查找：根据第 `i` 个位置取得元素。
- 按值查找：根据元素值寻找位置或结点。
- 插入：在指定位置加入新元素。
- 删除：删除指定位置或指定值的元素。
- 销毁：释放表占用的存储空间。

对本科阶段常见题目，重点通常在“插入、删除、查找、长度统计”的步骤和复杂度分析。

## 顺序存储与链式存储对比

顺序表：

- 使用连续存储空间，例如数组。
- 支持按下标随机访问。
- 插入和删除可能需要移动大量元素。
- 存储密度高，但容量固定或扩容成本较高。

链表：

- 使用结点和指针表示元素之间的关系。
- 不要求物理地址连续。
- 插入和删除时通常只需修改指针。
- 查找第 `i` 个元素时需要从头遍历。

## 逐步演示通用要求

当题目要求“演示”时，应写出每一步的数据状态变化。例如顺序表插入要展示移动前、每次后移、插入后的数组状态；链表插入要展示找到前驱、设置新结点指针、修改前驱指针后的链式关系。

状态表示建议：

- 顺序表：`[10, 20, 30]`
- 单链表：`head -> 10 -> 20 -> 30 -> NULL`
- 循环链表：`head -> 10 -> 20 -> 30 -> head`

## 时间复杂度总览

常见复杂度：

- 顺序表按位访问：`O(1)`
- 顺序表按值查找：`O(n)`
- 顺序表插入：平均 `O(n)`
- 顺序表删除：平均 `O(n)`
- 单链表按位查找：`O(n)`
- 单链表按值查找：`O(n)`
- 已知前驱结点的链表插入：`O(1)`
- 已知前驱结点的链表删除：`O(1)`
- 链表求长度：`O(n)`

## 易错点

- 把线性表的逻辑顺序误认为物理地址必须连续。
- 混淆“第 `i` 个元素”和“数组下标 `i`”。教材和算法描述常用 1 开始的位置序号，C 数组下标从 0 开始。
- 插入位置合法范围通常是 `1 <= i <= length + 1`。
- 删除位置合法范围通常是 `1 <= i <= length`。
- 链表操作时忘记处理空表、首元结点、尾结点等边界情况。
- 写 C 代码时忘记检查内存分配是否成功。
