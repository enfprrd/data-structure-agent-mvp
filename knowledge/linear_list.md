# 线性表

> 根据《数据结构：用 C 语言描述（第 2 版）》第 2 章整理的学习提纲，用于本项目 RAG 检索，不复制教材原文。

## 线性表概念

线性表是由相同类型数据元素组成的有限序列，通常记为 `L = (a1, a2, ..., an)`。`n` 是表长，`n = 0` 时为空表。

线性表的逻辑特点：
- 第一个元素没有直接前驱。
- 最后一个元素没有直接后继。
- 除第一个元素外，每个元素有且仅有一个直接前驱。
- 除最后一个元素外，每个元素有且仅有一个直接后继。

线性表只描述元素之间的一对一逻辑关系，不要求物理地址连续。它可以用顺序表实现，也可以用链表实现。

## 基本操作

常见操作包括：
- 初始化：建立一个空表。
- 判空：判断表中是否没有元素。
- 求长度：统计当前元素个数。
- 遍历：按逻辑顺序访问每个元素。
- 按位查找：取得第 `i` 个位置的元素。
- 按值查找：找到值等于给定值的元素位置或结点。
- 插入：在指定位置加入新元素。
- 删除：删除指定位置或指定值的元素。
- 销毁：释放表占用的存储空间。

## 顺序表与链表对比

顺序表：
- 使用一段连续存储空间，常用数组表示。
- 支持按下标随机访问，按位查找通常为 `O(1)`。
- 插入和删除可能需要移动大量元素。
- 存储密度高，但容量固定或扩容成本较高。

链表：
- 使用结点和指针表示元素之间的关系。
- 物理地址不要求连续。
- 已知前驱结点时插入、删除通常只需修改指针。
- 按位查找需要从头遍历，通常为 `O(n)`。

## 演示题通用要求

题目要求“演示”“逐步展示”时，应写出每一步的数据状态变化。

推荐状态表示：
- 顺序表：`[10, 20, 30]`
- 单链表：`head -> 10 -> 20 -> 30 -> NULL`
- 循环链表：`head -> 10 -> 20 -> 30 -> head`

讲解时不要只给最终结果，要说明为什么移动元素、为什么先改某个指针、边界条件是什么。

## 复杂度总览

- 顺序表按位访问：`O(1)`
- 顺序表按值查找：最好 `O(1)`，最坏 `O(n)`，平均 `O(n)`
- 顺序表插入：平均 `O(n)`
- 顺序表删除：平均 `O(n)`
- 单链表按位查找：`O(n)`
- 单链表按值查找：`O(n)`
- 已知前驱结点的链表插入：`O(1)`
- 已知前驱结点的链表删除：`O(1)`
- 链表求长度：`O(n)`

## 易错点

- 把线性表的逻辑顺序误认为物理地址必须连续。
- 混淆“第 `i` 个元素”和 C 数组下标 `i`。教材描述常用 1 开始的位置序号，C 数组下标从 0 开始。
- 插入位置合法范围通常是 `1 <= i <= length + 1`。
- 删除位置合法范围通常是 `1 <= i <= length`。
- 链表操作时忘记处理空表、首元结点、尾结点等边界情况。
