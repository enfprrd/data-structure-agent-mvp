# 串

> 根据教材第 4 章整理的学习提纲，用于本项目 RAG 检索，不复制教材原文。

## 串的概念

串是由零个或多个字符组成的有限序列，也称字符串。串中字符个数称为串长，长度为 0 的串称为空串。

串是一种特殊线性表，特殊之处在于每个元素都是字符，操作通常以子串为对象。

## 基本术语

- 主串：被查找或被处理的字符串。
- 子串：串中任意连续字符组成的序列。
- 位置：字符在串中的序号，教材描述常从 1 开始。
- 模式串：模式匹配中要寻找的字符串。

空串和空格串不同：空串长度为 0，空格串包含空格字符，长度不为 0。

## 存储方式

常见存储方式包括定长顺序存储、堆分配存储和块链存储。

C 语言字符串通常以字符数组表示，并用 `\0` 作为结束标记。教材中的串 ADT 可能显式保存长度，不能把两种约定混淆。

## 基本操作

常见操作包括赋值、复制、判空、求长度、比较、连接、求子串、插入、删除、清空、模式匹配。

串比较通常按字符编码逐个比较，先出现不同字符时由该字符决定大小；若前缀相同，长度较长者通常更大。

## 朴素模式匹配

目标：在主串 `S` 中查找模式串 `T` 第一次出现的位置。

基本思想：
1. 从主串某个位置开始，与模式串第一个字符对齐。
2. 逐个字符比较。
3. 若全部匹配，查找成功。
4. 若中途失配，主串起始位置后移一位，模式串重新从第一个字符比较。

最坏时间复杂度可达 `O(n*m)`，其中 `n` 是主串长度，`m` 是模式串长度。

## KMP 模式匹配

KMP 算法利用模式串自身的前后缀信息，在失配时移动模式串，避免主串指针回退。

核心是 `next` 或改进 `nextval` 数组：
- `next[j]` 表示模式串在第 `j` 个字符失配时，模式串指针应跳到的位置。
- 它只由模式串决定，与主串内容无关。

KMP 匹配时间复杂度通常为 `O(n + m)`。

## 易错点

- 混淆空串和空格串。
- 混淆 C 字符串的 `\0` 和教材串结构中的长度字段。
- 朴素匹配失配后，模式串应回到开头，主串起始位置后移一位。
- KMP 中 `next` 数组下标约定不同，写代码前必须统一从 0 还是从 1 开始。
- 误以为 `next` 数组依赖主串，实际上它只依赖模式串。
