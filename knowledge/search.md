# 查找

> 根据教材第 8 章整理的学习提纲，用于本项目 RAG 检索，不复制教材原文。

## 查找基本概念

查找是在数据集合中确定是否存在关键字等于给定值的记录。若找到，则称查找成功；否则查找失败。

基本术语：
- 查找表：由同一类型记录组成的集合。
- 关键字：能够标识记录的数据项。
- 静态查找表：只进行查找，不频繁插入删除。
- 动态查找表：查找过程中可能插入或删除记录。
- 平均查找长度 ASL：衡量查找效率的重要指标。

## 顺序查找

顺序查找从表的一端开始逐个比较关键字。

特点：
- 对顺序表和链表都适用。
- 对表是否有序没有要求。
- 成功查找平均比较次数通常为 `O(n)`，失败也为 `O(n)`。

可设置哨兵减少循环中的边界判断。

## 折半查找

折半查找适用于有序顺序表。

基本过程：
1. 设定查找区间 `low` 和 `high`。
2. 取中间位置 `mid`。
3. 若 `key == data[mid]`，查找成功。
4. 若 `key < data[mid]`，在左半区继续。
5. 若 `key > data[mid]`，在右半区继续。

时间复杂度：`O(log n)`。

前提条件：表必须有序，且能随机访问。因此链表不适合直接做折半查找。

## 分块查找

分块查找又称索引顺序查找。将表分成若干块，块间有序、块内可以无序。先查索引确定目标块，再在块内顺序查找。

## 二叉排序树

二叉排序树又称二叉查找树。对任意结点：左子树关键字小于该结点，右子树关键字大于该结点。

中序遍历二叉排序树可以得到有序序列。

查找、插入、删除平均效率与树高有关。若树接近平衡，效率约为 `O(log n)`；若退化成单支树，最坏为 `O(n)`。

## 平衡二叉树

平衡二叉树通过旋转保持左右子树高度差受限，避免二叉排序树退化。

常见调整包括单旋和双旋。旋转后仍要保持二叉排序树的中序有序性。

## B 树与 B+ 树

B 树是多路平衡查找树，适合外存查找。每个结点可保存多个关键字和多个孩子，从而降低树高，减少磁盘访问次数。

B+ 树常用于数据库和文件系统。其数据记录通常集中在叶子层，叶子结点之间可顺序链接，适合范围查询。

## 散列表

散列表通过散列函数把关键字映射到存储地址，理想情况下查找接近 `O(1)`。

常见冲突处理方法：
- 开放定址法：冲突后继续探测其他位置。
- 链地址法：同一地址的记录组织成链表。

## 易错点

- 折半查找必须基于有序顺序表。
- 二叉排序树删除有两个孩子的结点时，常用中序前驱或中序后继替代。
- 平衡调整后仍必须满足二叉排序树性质。
- B 树和二叉树不是同一类结构，B 树是多路查找树。
- 散列查找不是永远 `O(1)`，冲突严重时效率会下降。
