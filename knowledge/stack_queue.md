# 栈和队列

> 根据教材第 3 章整理的学习提纲，用于本项目 RAG 检索，不复制教材原文。

## 限定性线性表

栈和队列都是操作受限制的线性表。它们的特殊性不在元素类型，而在插入和删除的位置受到限制。

栈只允许在同一端进行插入和删除；队列通常在一端插入、另一端删除。

## 栈的概念

栈是只允许在表的一端进行插入和删除的线性表。允许操作的一端称为栈顶，另一端称为栈底。

特点：后进先出，记作 LIFO。

基本操作：
- 初始化栈。
- 判空。
- 入栈 `push`：把新元素压入栈顶。
- 出栈 `pop`：删除并返回栈顶元素。
- 取栈顶 `getTop`：读取栈顶元素但不删除。

## 顺序栈

顺序栈用数组保存元素，用 `top` 表示栈顶位置。

常见约定一：`top == -1` 表示空栈，入栈先 `top++` 再写入，出栈先读取再 `top--`。

判空：
```c
top == -1
```

判满：
```c
top == MAXSIZE - 1
```

入栈和出栈时间复杂度均为 `O(1)`。

## 队列的概念

队列是只允许在一端插入、另一端删除的线性表。允许插入的一端称为队尾，允许删除的一端称为队头。

特点：先进先出，记作 FIFO。

基本操作：
- 初始化队列。
- 判空。
- 入队 `enqueue`：从队尾加入元素。
- 出队 `dequeue`：从队头删除并返回元素。
- 取队头元素。

## 循环队列

普通顺序队列出队后会产生“假溢出”：数组前面有空位，但队尾已经到达数组末端。循环队列把数组看作首尾相接，用取模运算移动指针。

常见约定：牺牲一个存储单元区分空和满。

判空：
```c
front == rear
```

判满：
```c
(rear + 1) % MAXSIZE == front
```

入队：
```c
data[rear] = x;
rear = (rear + 1) % MAXSIZE;
```

出队：
```c
x = data[front];
front = (front + 1) % MAXSIZE;
```

循环队列入队、出队时间复杂度均为 `O(1)`。

## 链队列

链队列通常设置队头指针 `front` 和队尾指针 `rear`。带头结点时，空队列满足 `front == rear`。

入队时把新结点接到队尾，并移动 `rear`。出队时删除队头后的第一个数据结点；如果删除后队列为空，要让 `rear` 回到头结点。

## 应用

栈的典型应用：括号匹配、表达式求值、函数调用、递归、深度优先搜索、进制转换。

队列的典型应用：排队服务、缓冲区、广度优先搜索、树的层序遍历。

## 栈的经典应用

栈的核心思想是后进先出，适合保存“尚未处理完、需要回退或逆序处理”的信息。常见教学与算法示例包括：

- 基础操作：顺序栈入栈、顺序栈出栈、链栈入栈、链栈出栈。
- 进制转换：不断把余数入栈，再逆序出栈得到目标进制表示。
- 括号匹配：左括号入栈，右括号与栈顶匹配后出栈。
- 表达式处理：中缀表达式转后缀表达式、后缀表达式求值、前缀表达式求值、运算符优先级表达式求值。
- 递归与调用：函数调用栈、递归调用过程、汉诺塔递归过程、汉诺塔非递归栈模拟。
- 回溯与搜索：迷宫求解、回溯路径保存、非递归深度优先搜索 DFS。
- 树遍历：二叉树先序、中序、后序遍历的非递归实现。
- 实际系统应用：浏览器前进后退、文本编辑器撤销重做、XML/HTML 标签嵌套检查、编译器语法分析栈。
- 算法题应用：单调栈、下一个更大元素、柱状图最大矩形、每日温度。
- 栈与队列综合：用两个栈实现队列、栈逆序、栈排序、火车调度、判断出栈序列是否合法。

这些例子虽然问题外观不同，但共同点都是利用栈保存“最近遇到且最先需要处理”的对象。

## 易错点

- 混淆栈顶和栈底，导致入栈、出栈方向不一致。
- 顺序栈忘记判断栈满，或空栈仍执行出栈。
- 循环队列判满公式写错。
- 链队列删除最后一个元素后忘记修改 `rear`。
- 把队列的 FIFO 特性写成 LIFO。
