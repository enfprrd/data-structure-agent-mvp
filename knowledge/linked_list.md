# 链表

> 根据教材第 2 章链式存储部分整理的学习提纲，用于本项目 RAG 检索，不复制教材原文。

## 单链表概念

单链表是线性表的链式存储实现。每个元素存放在一个结点中，结点通常包含数据域和指针域。

常见结点定义：
```c
typedef struct LNode {
    int data;
    struct LNode *next;
} LNode, *LinkList;
```

`data` 保存数据元素，`next` 指向后继结点。单链表可以带头结点，也可以不带头结点。初学和课堂演示建议优先使用带头结点写法，因为插入和删除首元结点时更统一。

带头结点空表示例：
```text
head -> NULL
```

## 按位查找

目标：找到第 `i` 个数据结点。

基本步骤：
1. 指针 `p` 从首元结点开始。
2. 计数器 `j` 从 1 开始。
3. 当 `p != NULL` 且 `j < i` 时，执行 `p = p->next`，`j++`。
4. 若最后 `p != NULL`，则找到第 `i` 个结点；否则查找失败。

时间复杂度：`O(n)`。

## 按值查找

目标：查找值等于 `x` 的结点。

基本步骤：
1. 从首元结点开始。
2. 逐个比较 `p->data` 与 `x`。
3. 相等则返回该结点指针。
4. 走到 `NULL` 表示查找失败。

复杂度：最好 `O(1)`，最坏 `O(n)`，平均 `O(n)`。

## 长度统计

从首元结点开始遍历，每访问一个数据结点计数加 1，直到 `NULL`。

示例：
```text
head -> 10 -> 20 -> 30 -> NULL
访问 10：count = 1
访问 20：count = 2
访问 30：count = 3
```

时间复杂度：`O(n)`。

## 单链表插入

目标：在第 `i` 个位置插入新元素 `x`。带头结点时通常先找到第 `i - 1` 个结点，也就是插入位置的前驱结点 `p`。

合法位置：
```text
1 <= i <= length + 1
```

核心指针操作：
```c
s->next = p->next;
p->next = s;
```

必须先让新结点 `s` 指向原后继，再让前驱 `p` 指向 `s`。如果先执行 `p->next = s`，原后继地址可能丢失。

示例：在 `20` 和 `30` 之间插入 `99`。
```text
初始：head -> 10 -> 20 -> 30 -> NULL
找到前驱 p：p 指向 20
创建新结点 s：s 存放 99
执行 s->next = p->next：99 -> 30
执行 p->next = s：20 -> 99
结果：head -> 10 -> 20 -> 99 -> 30 -> NULL
```

复杂度：查找前驱 `O(n)`，已知前驱后插入 `O(1)`。

## 单链表删除

目标：删除第 `i` 个位置的结点。带头结点时先找到第 `i - 1` 个结点，即待删结点的前驱 `p`。

核心指针操作：
```c
q = p->next;
p->next = q->next;
free(q);
```

`q` 保存待删结点，避免直接改指针后无法释放内存。

示例：删除 `20`。
```text
初始：head -> 10 -> 20 -> 30 -> NULL
找到前驱 p：p 指向 10
q = p->next：q 指向 20
p->next = q->next：10 -> 30
释放 q
结果：head -> 10 -> 30 -> NULL
```

复杂度：查找前驱 `O(n)`，已知前驱后删除 `O(1)`。

## 头插法与尾插法建表

头插法：
- 每次把新结点插入到头结点之后。
- 得到的链表元素顺序与输入顺序相反。
- 适合需要逆序建表的场景。

尾插法：
- 使用尾指针 `tail` 指向当前最后一个结点。
- 每次把新结点接到尾部。
- 得到的链表元素顺序与输入顺序一致。

## 循环链表

循环链表的尾结点指针不为空，而是指向头结点或首元结点，从而形成环。

遍历循环链表时不能用 `p != NULL` 作为结束条件，应判断是否回到头结点或起始结点。

示例：
```c
while (p != head) {
    p = p->next;
}
```

## 双向链表

双向链表的每个结点通常含有 `prior` 和 `next` 两个指针，既能找后继，也能找前驱。

插入结点时要同时维护前驱和后继方向的指针。删除结点时也要改两侧结点的指针。双向链表操作更方便，但每个结点需要额外指针空间。

## 易错点

- 插入时先改 `p->next`，导致原后继结点地址丢失。
- 删除时忘记保存待删结点 `q`，导致无法释放内存。
- 空表时访问 `head->next->data`，造成空指针错误。
- 不带头结点的链表删除首元结点时，忘记单独修改头指针。
- 循环链表遍历仍用 `p != NULL`，导致死循环或逻辑错误。
- 使用 `malloc` 后没有判断是否为 `NULL`。
