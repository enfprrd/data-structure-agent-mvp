# 链表

> 说明：本文件是面向 RAG 的本科数据结构学习摘要，不复制教材原文。链表部分后续可继续补充教材中的图示、算法描述和习题。

## 单链表概念

单链表是线性表的链式存储实现。每个元素存放在一个结点中，结点通常包含数据域和指针域。

常见 C 结点定义：

```c
typedef struct LNode {
    int data;
    struct LNode *next;
} LNode, *LinkList;
```

其中：

- `data` 保存数据元素。
- `next` 指向后继结点。
- `LinkList` 通常表示指向结点的指针。

单链表可以带头结点，也可以不带头结点。本 MVP 建议优先讲带头结点的写法，因为插入和删除首元结点时更统一。

带头结点示例：

```text
head -> 10 -> 20 -> 30 -> NULL
```

`head` 不存放有效数据，只用于统一操作。

## 单链表初始化

带头结点单链表初始化的基本思想：

1. 为头结点申请内存。
2. 判断内存申请是否成功。
3. 将头结点的 `next` 置为 `NULL`。

空表状态：

```text
head -> NULL
```

## 单链表按位查找

目标：找到第 `i` 个数据结点。

合法位置通常从 `1` 开始。

基本步骤：

1. 指针 `p` 从首元结点开始。
2. 计数器 `j` 从 `1` 开始。
3. 当 `p != NULL` 且 `j < i` 时，`p = p->next`，`j++`。
4. 若最后 `p != NULL`，则找到第 `i` 个结点；否则查找失败。

时间复杂度：`O(n)`。

## 单链表按值查找

目标：查找值等于 `x` 的结点。

基本步骤：

1. 从首元结点开始。
2. 逐个比较 `p->data` 与 `x`。
3. 若相等，返回该结点指针。
4. 若走到 `NULL`，表示查找失败。

时间复杂度：最好 `O(1)`，最坏 `O(n)`，平均 `O(n)`。

## 链表长度统计

目标：统计数据结点个数。

基本步骤：

1. `count = 0`。
2. `p` 指向首元结点。
3. 每访问一个数据结点，`count++`。
4. `p = p->next`。
5. 当 `p == NULL` 时结束。

示例：

```text
head -> 10 -> 20 -> 30 -> NULL
count 初始为 0
访问 10，count = 1
访问 20，count = 2
访问 30，count = 3
结束，长度为 3
```

时间复杂度：`O(n)`。

## 单链表插入

目标：在第 `i` 个位置插入新元素 `x`。带头结点时，通常先找到第 `i - 1` 个结点，也就是插入位置的前驱结点。

合法位置：

```text
1 <= i <= length + 1
```

核心指针操作：

```c
s->next = p->next;
p->next = s;
```

其中：

- `p` 是前驱结点。
- `s` 是新结点。

顺序不能轻易写反。如果先执行 `p->next = s`，原来的后继结点地址可能丢失。

示例：在 `20` 和 `30` 之间插入 `99`。

```text
初始：head -> 10 -> 20 -> 30 -> NULL
找到前驱 p：p 指向 20
创建新结点 s：s 存放 99
执行 s->next = p->next：99 -> 30
执行 p->next = s：20 -> 99
结果：head -> 10 -> 20 -> 99 -> 30 -> NULL
```

时间复杂度：

- 查找前驱结点：`O(n)`。
- 已知前驱结点后的插入：`O(1)`。
- 按位置插入整体：`O(n)`。

## 单链表删除

目标：删除第 `i` 个位置的结点。带头结点时，通常先找到第 `i - 1` 个结点，即待删结点的前驱结点。

核心指针操作：

```c
q = p->next;
p->next = q->next;
free(q);
```

其中：

- `p` 是前驱结点。
- `q` 是待删除结点。

示例：删除 `20`。

```text
初始：head -> 10 -> 20 -> 30 -> NULL
找到前驱 p：p 指向 10
令 q = p->next：q 指向 20
执行 p->next = q->next：10 -> 30
释放 q
结果：head -> 10 -> 30 -> NULL
```

时间复杂度：

- 查找前驱结点：`O(n)`。
- 已知前驱结点后的删除：`O(1)`。
- 按位置删除整体：`O(n)`。

## 头插法和尾插法建表

头插法：

- 每次把新结点插入到头结点之后。
- 得到的链表元素顺序与输入顺序相反。
- 插入操作简单，适合需要逆序建表的场景。

尾插法：

- 使用尾指针 `tail` 指向当前最后一个结点。
- 每次把新结点接到尾部。
- 得到的链表元素顺序与输入顺序一致。

## 循环链表

循环链表是一种尾结点指针不为空的链表。它的尾结点通常指向头结点或首元结点，从而形成环。

带头结点循环单链表示例：

```text
head -> 10 -> 20 -> 30
 ^                  |
 |__________________|
```

判断循环链表遍历结束时，不能用 `p == NULL`，而应判断是否回到头结点或起始结点。

例如带头结点循环链表遍历条件常写为：

```c
while (p != head) {
    ...
    p = p->next;
}
```

循环链表优点：

- 从任一结点出发都可以继续访问后续结点。
- 适合需要循环处理的场景。
- 若只设尾指针，访问头结点和尾结点都比较方便。

## 链表 C 代码要点

常用函数接口可以设计为：

```c
LinkList InitList(void);
int ListLength(LinkList head);
LNode *GetElem(LinkList head, int i);
LNode *LocateElem(LinkList head, int x);
int ListInsert(LinkList head, int i, int x);
int ListDelete(LinkList head, int i, int *deleted);
void PrintList(LinkList head);
void DestroyList(LinkList head);
```

写完整可运行 C 程序时，应包含：

- `#include <stdio.h>`
- `#include <stdlib.h>`
- 结点结构定义
- 初始化函数
- 插入或删除等核心函数
- 打印函数
- 释放链表函数
- `main` 中的基础测试

## 易错点

- 插入时先改 `p->next`，导致原后继结点地址丢失。
- 删除时忘记保存待删结点 `q`，导致无法释放内存。
- 删除尾结点时没有正确处理 `q->next == NULL` 的情况。
- 空表时访问 `head->next->data`，导致空指针错误。
- 不带头结点链表删除首元结点时需要单独处理头指针变化。
- 循环链表遍历时仍然用 `p != NULL`，导致死循环或逻辑错误。
- 使用 `malloc` 后没有判断返回值是否为 `NULL`。
- 删除结点后继续使用已经释放的指针。
