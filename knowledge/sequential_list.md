# 顺序表

> 根据教材第 2 章顺序存储部分整理的学习提纲，用于本项目 RAG 检索，不复制教材原文。

## 顺序表概念

顺序表是线性表的顺序存储实现，用一组地址连续的存储单元依次存放线性表元素，常用 C 数组表示。

常见结构：
```c
#define MAXSIZE 100

typedef struct {
    int data[MAXSIZE];
    int length;
} SqList;
```

其中 `data` 保存元素，`length` 表示当前元素个数，`MAXSIZE` 表示最大容量。

## 初始化、判空与判满

初始化通常令 `length = 0`。

判空条件：
```c
L.length == 0
```

判满条件：
```c
L.length == MAXSIZE
```

顺序表所有插入操作都要先考虑容量是否已满，否则可能数组越界。

## 按位查找

目标：取得第 `i` 个元素。

合法范围：
```text
1 <= i <= length
```

若位置序号从 1 开始，则第 `i` 个元素在 C 数组中的下标是 `i - 1`。

时间复杂度：`O(1)`，因为可以按下标直接访问。

## 按值查找

目标：查找值等于 `x` 的元素位置。

基本过程：
1. 从下标 `0` 开始顺序扫描。
2. 若 `data[k] == x`，返回位置 `k + 1`。
3. 扫描结束仍未找到，返回失败标记，例如 `0` 或 `-1`。

复杂度：最好 `O(1)`，最坏 `O(n)`，平均 `O(n)`。

## 插入操作

目标：在第 `i` 个位置插入元素 `x`。

合法条件：
```text
1 <= i <= length + 1
length < MAXSIZE
```

核心步骤：
1. 判断表是否已满。
2. 判断插入位置是否合法。
3. 从最后一个元素开始，到第 `i` 个元素为止，依次向后移动一位。
4. 将 `x` 放入下标 `i - 1`。
5. `length` 加 1。

示例：在 `[10, 20, 30, 40]` 的第 3 个位置插入 `99`。
```text
初始：[10, 20, 30, 40]
移动 40：[10, 20, 30, 40, 40]
移动 30：[10, 20, 30, 30, 40]
写入 99：[10, 20, 99, 30, 40]
```

复杂度：尾部插入最好 `O(1)`，头部插入最坏 `O(n)`，平均 `O(n)`。

## 删除操作

目标：删除第 `i` 个位置的元素。

合法条件：
```text
1 <= i <= length
```

核心步骤：
1. 判断删除位置是否合法。
2. 保存被删除元素，便于返回或输出。
3. 从第 `i + 1` 个元素开始，依次向前移动一位。
4. `length` 减 1。

示例：删除 `[10, 20, 30, 40, 50]` 的第 2 个元素。
```text
初始：[10, 20, 30, 40, 50]
移动 30：[10, 30, 30, 40, 50]
移动 40：[10, 30, 40, 40, 50]
移动 50：[10, 30, 40, 50, 50]
长度减 1：[10, 30, 40, 50]
```

复杂度：删除尾元素最好 `O(1)`，删除首元素最坏 `O(n)`，平均 `O(n)`。

## C 代码要点

常见接口：
```c
int ListInsert(SqList *L, int i, int x);
int ListDelete(SqList *L, int i, int *deleted);
```

建议成功返回 `1`，失败返回 `0`。传入指针是为了在函数内修改顺序表本身。

## 易错点

- 插入时移动方向写反。必须从后往前移动，避免覆盖尚未移动的元素。
- 删除时移动范围写错。应从被删位置的后一个元素开始往前移。
- 忘记判断表满导致数组越界。
- 忘记判断位置是否合法。
- 删除后只移动元素，却忘记 `length--`。
