# 顺序表

> 说明：本文件是面向 RAG 的本科数据结构学习摘要，不复制教材原文。顺序表部分后续可继续补充教材中的类型定义、例题和课后习题。

## 顺序表概念

顺序表是线性表的顺序存储实现。它使用一组地址连续的存储单元依次存放线性表元素，常用 C 数组表示。

常见 C 结构设计：

```c
#define MAXSIZE 100

typedef struct {
    int data[MAXSIZE];
    int length;
} SqList;
```

其中：

- `data` 保存元素。
- `length` 表示当前元素个数。
- `MAXSIZE` 表示最大容量。

顺序表的核心特点：

- 逻辑相邻的元素在物理位置上也相邻。
- 可以通过下标直接访问第 `i` 个元素。
- 插入、删除时需要移动元素来保持连续存储。

## 顺序表初始化和判空

初始化通常将 `length` 置为 0。

判空条件：

```c
L.length == 0
```

判满条件：

```c
L.length == MAXSIZE
```

## 顺序表按位查找

按位查找第 `i` 个元素时，应先判断位置是否合法。

位置合法范围：

```text
1 <= i <= length
```

若采用 1 开始的位置序号，则第 `i` 个元素在 C 数组中的下标是 `i - 1`。

时间复杂度：`O(1)`。

## 顺序表按值查找

按值查找需要从前到后依次比较。

基本步骤：

1. 从下标 `0` 开始扫描。
2. 若 `data[k] == target`，返回位置 `k + 1`。
3. 扫描结束仍未找到，返回失败标记，例如 `0` 或 `-1`。

时间复杂度：最好 `O(1)`，最坏 `O(n)`，平均 `O(n)`。

## 顺序表插入

目标：在第 `i` 个位置插入元素 `x`。

合法条件：

```text
1 <= i <= length + 1
length < MAXSIZE
```

核心步骤：

1. 判断表是否已满。
2. 判断插入位置是否合法。
3. 从最后一个元素开始，到第 `i` 个元素为止，依次向后移动一位。
4. 将 `x` 放入下标 `i - 1`。
5. `length` 加 1。

示例：在 `[10, 20, 30, 40]` 的第 3 个位置插入 `99`。

```text
初始状态：[10, 20, 30, 40]
移动 40：[10, 20, 30, 40, 40]
移动 30：[10, 20, 30, 30, 40]
插入 99：[10, 20, 99, 30, 40]
```

时间复杂度：

- 最好情况：尾部插入，不移动元素，`O(1)`。
- 最坏情况：表头插入，移动 `n` 个元素，`O(n)`。
- 平均情况：`O(n)`。

## 顺序表删除

目标：删除第 `i` 个位置的元素。

合法条件：

```text
1 <= i <= length
```

核心步骤：

1. 判断删除位置是否合法。
2. 保存被删除元素，便于返回或输出。
3. 从第 `i + 1` 个元素开始，依次向前移动一位。
4. `length` 减 1。

示例：删除 `[10, 20, 30, 40, 50]` 的第 2 个元素。

```text
初始状态：[10, 20, 30, 40, 50]
移动 30：[10, 30, 30, 40, 50]
移动 40：[10, 30, 40, 40, 50]
移动 50：[10, 30, 40, 50, 50]
长度减 1：[10, 30, 40, 50]
```

时间复杂度：

- 最好情况：删除尾元素，不移动元素，`O(1)`。
- 最坏情况：删除首元素，移动 `n - 1` 个元素，`O(n)`。
- 平均情况：`O(n)`。

## 顺序表插入删除 C 代码要点

顺序表插入函数常见接口：

```c
int ListInsert(SqList *L, int i, int x);
```

顺序表删除函数常见接口：

```c
int ListDelete(SqList *L, int i, int *deleted);
```

返回值建议：

- `1` 表示成功。
- `0` 表示失败。

## 易错点

- 插入时移动方向写反。必须从后往前移动，否则会覆盖还没移动的元素。
- 删除时移动范围写错。应从被删位置的后一个元素开始往前移。
- 忘记判断表满导致数组越界。
- 忘记判断位置合法。
- 混淆位置序号和数组下标。
- 删除后只移动了元素，却忘记 `length--`。
